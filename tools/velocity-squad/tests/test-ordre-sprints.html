🧪 Test - Ordre d'Affichage des Sprints

📋 Objectif du Test

Vérifier que les sprints s'affichent dans l'ordre chronologique : du plus ancien (gauche) au plus récent (droite).

📊 Ordre Attendu

✅ Comportement Correct

Les sprints doivent s'afficher de gauche à droite dans l'ordre chronologique :

Sprint 1
(Plus ancien)
→
Sprint 2
→
Sprint 3
→
Sprint 4
(Plus récent)

Lecture naturelle : Le temps progresse de gauche à droite, comme une timeline.

🔍 Tests à Effectuer

Test 1 : Vue Vélocité (📊)

  • Ouvrir l'application avec plusieurs sprints
  • Vérifier que le sprint le plus ancien est à gauche
  • Vérifier que le sprint le plus récent est à droite
  • Vérifier que l'ordre est chronologique

Test 2 : Vue Burndown (📉)

  • Changer vers la vue Burndown
  • Vérifier que l'ordre est identique à la vue Vélocité
  • Vérifier que les labels correspondent aux bons sprints

Test 3 : Vue Burnup (📈)

  • Changer vers la vue Burnup
  • Vérifier que l'ordre est identique aux autres vues
  • Vérifier la cohérence des données

Test 4 : Annotations

  • Ajouter une annotation sur un sprint spécifique
  • Vérifier que l'icône apparaît au bon endroit
  • Survoler l'annotation : vérifier le tooltip
  • Cliquer sur l'annotation : vérifier les détails

Test 5 : Tooltips

  • Survoler chaque barre/point du graphique
  • Vérifier que le nom du sprint correspond
  • Vérifier que les données affichées sont correctes
  • Vérifier que les annotations s'affichent dans le tooltip

🔧 Vérification Technique

Console du Navigateur

Ouvrir la console (F12) et exécuter :

// Afficher l'ordre des sprints dans les données console.log('Ordre original:', window.velocityTool.data.sprints.map(s => s.name) ); // Afficher l'ordre inversé (affiché) console.log('Ordre affiché:', [...window.velocityTool.data.sprints].reverse().map(s => s.name) );

Résultat Attendu

// Exemple avec 4 sprints Ordre original: ["Sprint 4", "Sprint 3", "Sprint 2", "Sprint 1"] Ordre affiché: ["Sprint 1", "Sprint 2", "Sprint 3", "Sprint 4"]

📝 Scénarios de Test

Scénario 1 : Ajout d'un Nouveau Sprint

  1. Ajouter un nouveau sprint (Sprint 5)
  2. Vérifier qu'il apparaît à droite (plus récent)
  3. Vérifier que les autres sprints restent dans l'ordre

Scénario 2 : Import de Données

  1. Exporter les données actuelles
  2. Importer un fichier avec des sprints dans le désordre
  3. Vérifier que l'affichage est chronologique

Scénario 3 : Changement de Vue

  1. Noter l'ordre dans la vue Vélocité
  2. Changer vers Burndown puis Burnup
  3. Vérifier que l'ordre reste identique

Scénario 4 : Annotations Multiples

  1. Ajouter 3 annotations sur des sprints différents
  2. Vérifier que chaque annotation est au bon endroit
  3. Changer de vue : vérifier que les annotations suivent

✅ Checklist de Validation

  • Sprint le plus ancien à gauche
  • Sprint le plus récent à droite
  • Ordre chronologique respecté
  • Cohérence entre les 3 vues
  • Annotations correctement positionnées
  • Tooltips affichent les bonnes données
  • Interactions (hover, clic) fonctionnent
  • Ajout de sprint : apparaît à droite
  • Import/Export : ordre préservé
  • Pas d'erreur dans la console

🐛 Problèmes Potentiels

❌ Ordre Inversé

Symptôme : Le sprint le plus récent est à gauche

Cause : L'inversion n'est pas appliquée

Solution : Vérifier que [...this.data.sprints].reverse() est bien utilisé

❌ Annotations Décalées

Symptôme : Les icônes ne sont pas au bon endroit

Cause : L'index n'est pas inversé dans le plugin

Solution : Vérifier le calcul de displayIndex dans le plugin

❌ Tooltips Incorrects

Symptôme : Le tooltip affiche les données d'un autre sprint

Cause : L'index n'est pas inversé dans les callbacks

Solution : Vérifier le calcul dans afterBody
